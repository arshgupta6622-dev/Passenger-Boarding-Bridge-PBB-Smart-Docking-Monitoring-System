# Passenger Boarding Bridge (PBB) Smart Docking & Monitoring System

An advanced system for efficient docking, operation, and monitoring of Passenger Boarding Bridges (PBBs / jet-bridges) across an airport's gate infrastructure. Ensures safe aircraft–gate interface, optimal turnaround time, and improved utilisation of gate assets.

**Developed in association with Swati Airport Services** as a software module for real-time PBB monitoring and performance optimisation.

## What it does

Given a day's flight schedule (arrival, departure, aircraft type), the system:

1. **Automatically docks each flight** to a compatible Passenger Boarding Bridge using a first-fit scheduling algorithm with a 15-minute turnaround buffer.
2. **Enforces safety protocols** — wide-body aircraft are never assigned to narrow-body-only PBBs; remote stands are distinguished from jet-bridge gates.
3. **Monitors utilisation in real time** via an interactive Gantt timeline, colour-coded per airline with hover-detail.
4. **Tracks key metrics** — overall utilisation %, per-bridge utilisation, peak-hour detection, average turnaround time, jet-bridge vs remote-stand share, conflicts, unassigned count.
5. **Surfaces conflicts** with plain-English explanations generated by Gemini.
6. **Proposes optimisations** — an AI-generated set of bridge swaps that improve utilisation, reviewed and applied one-by-one.

## Quick start

```bash
npm install
cp .env.example .env.local
# paste your Gemini API key into .env.local
npm run dev
```

Open http://localhost:3000.

## Features

- First-fit scheduler with 15-minute turnaround buffer and aircraft/PBB compatibility rules
- Interactive Gantt timeline: 24 h × 8 bridges, airline colour-coding, hover tooltips, smooth Framer Motion animations
- 5 Gemini-powered AI modules:
  - **Natural-language flight entry** — paste `AI 202 from Delhi at 10:30, wide body, departs 12:15` and the form auto-fills
  - **Bulk import** — Gemini normalises a messy airline timetable or CSV into structured rows
  - **Conflict explanation** — plain-English reason for any unassigned flight + suggested action
  - **AI Optimise** — proposes PBB reassignments to lift utilisation; review and apply individually
  - **What-if chat** — `What happens if AI 101 is delayed 30 minutes?` returns a scenario walkthrough
- Metrics dashboard with 6 live cards + per-gate utilisation bar chart, before/after deltas after each schedule run
- Dark mode by default · localStorage persistence · no database
